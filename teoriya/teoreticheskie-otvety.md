## 1. История и основы

### Вопрос 1. История развития вычислительной техники по поколениям

| Поколение | Период | Элементная база | Характеристика |
|-----------|--------|-----------------|----------------|
| **1** | 1940–1950-е | Электронные лампы | ENIAC, огромные размеры, низкая надёжность, программы на переключателях и карточках |
| **2** | 1950–1960-е | Транзисторы | Уменьшение размеров, появление языков программирования (Fortran, Cobol), магнитные ленты и диски |
| **3** | 1960–1970-е | Интегральные схемы (ИС) | Мини-ЭВМ, операционные системы, развитие сетей |
| **4** | 1970–1980-е | Большие и сверхбольшие ИС (БИС, СБИС) | Персональные компьютеры, микропроцессоры (Intel 4004, 8080), массовое распространение |
| **5** | 1990-е — н.в. | Сверхбольшие ИС, параллельные архитектуры | Многоядерность, сети, облачные вычисления, встраиваемые системы |

**Ключевая идея для экзамена:** каждое поколение определяется прежде всего **элементной базой**, от которой зависят скорость, стоимость, размер и область применения.

## 2. Микропроцессор: понятие и классификация

### Вопрос 2. Определение «Микропроцессор». Параметры

**Микропроцессор (МП)** — программно-управляемое большое интегральное устройство, выполняющее функции центрального процессора: обработку данных, выполнение команд, управление устройствами системы. Реализован на одном или нескольких кристаллах.

**Основные параметры:**

| Параметр | Описание |
|----------|----------|
| Разрядность | Число бит в слове данных и адреса |
| Тактовая частота | Число тактов в секунду |
| Система команд | Набор и форматы инструкций |
| Быстродействие | MIPS, MFLOPS |
| Техпроцесс | Норма литографии (нм), энергопотребление |
| Корпус и выводы | DIP, QFP, BGA и т.д. |
| Набор периферии | Кэш, FPU, встроенные контроллеры |

### Вопрос 3. Классификация по назначению и виду информации

**По назначению:**
- **Универсальные** — ПК, серверы (Intel Core, AMD Ryzen)
- **Специализированные** — цифровая обработка сигналов (DSP), графика (GPU), управление (МК)
- ** Отраслевые** — для телекоммуникаций, автомобилей, медицины

**По виду обрабатываемой информации:**
- **Цифровые** — дискретные данные (большинство современных МП)
- **Аналоговые** — непрерывные сигналы (редко; чаще отдельные САП)
- **Цифро-аналоговые гибриды**

### Вопрос 4. Классификация по разрядности и тактовой частоте

**По разрядности:** 4, 8, 16, 32, 64 бита (ширина шины данных и регистров).

**По тактовой частоте:**
- Низкочастотные (кГц–МГц) — встраиваемые, энергосберегающие
- Среднечастотные (десятки–сотни МГц) — МК, промышленные контроллеры
- Высокочастотные (ГГц) — настольные и серверные процессоры

### Вопрос 5. Классификация по синхронизации и компоновке

**По синхронизации:**
- **Синхронные** — все операции привязаны к тактовому сигналу
- **Асинхронные** — без единого глобального такта (исторически; сейчас редкость)

**По компоновке:**
- Монолитные (всё на одном кристалле)
- Многокристальные (CPU + сопроцессор + кэш отдельно)
- Многокристальные модули (Multi-Chip Module)

### Вопрос 6. Классификация по магистралям и системе команд

**По числу управляющих магистралей:**
- Одномагистральные (общая шина адреса/данных/управления)
- Многомагистральные (раздельные шины)

**По системе команд:**
- CISC, RISC, VLIW, MISC (см. раздел 5)

## 3. Архитектуры ЭВМ

### Вопрос 7. Принстонская (фон Неймана) архитектура

**Суть:** единое адресное пространство для программ и данных. Команды и данные хранятся в одной памяти.

```
    ┌──────────┐
    │   CPU    │
    │ (АЛУ+УУ) │
    └────┬─────┘
         │ Общая шина
    ┌────┴─────┐
    │  Память  │  ← команды + данные
    └──────────┘
         │
    ┌────┴─────┐
    │   В/В    │
    └──────────┘
```

**Достоинства:** простота, гибкость (программа как данные), универсальность.  
**Недостатки:** «узкое горлышко» фон Неймана — одна шина для команд и данных ограничивает пропускную способность.

### Вопрос 8. Гарвардская архитектура

**Суть:** раздельные память и шины для команд и данных.

```
    ┌──────────┐
    │   CPU    │
    └──┬────┬──┘
       │    │
  Шина │    │ Шина
  команд   данных
       │    │
  ┌────┴┐ ┌─┴────┐
  │ПЗУ/ │ │  ОЗУ │
  │Flash│ │      │
  └─────┘ └──────┘
```

**Достоинства:** параллельный доступ к команде и данным, выше скорость (МК PIC, AVR, ARM Cortex-M).  
**Недостатки:** сложнее структура, раздельные объёмы памяти программ и данных.

## 4. Тактовая частота, MIPS, разрядность

### Вопрос 9. Тактовая частота и MIPS

**Тактовая частота** — число тактов генератора в секунду (Гц). Один такт — минимальный интервал синхронизации.

**MIPS** (Million Instructions Per Second) — миллионов команд в секунду. Показатель производительности.

**Эволюция сокращения тактов на команду:**
- Ранние МП: несколько тактов на команду
- Конвейер: 1 команда за такт в идеале (CPI → 1)
- Суперскалярность: >1 команды за такт
- Важно: рост частоты ≠ пропорциональный рост MIPS из-за CPI, кэша, ветвлений

### Вопрос 10. Разрядность процессора

**Разрядность** — число бит, обрабатываемых за одну операцию (регистры, шина данных, иногда адреса).

**Маркировка:** 8080 (8 бит), 8086 (16), 80386 (32), Core i7 (64). Буквы в индексе (SX, DX, LX) часто указывают на шину/корпус.

### Вопрос 11. Система команд. Микро- и макроархитектура

**Система команд** — совокупность всех команд МП, их форматов, способов адресации и правил выполнения.

- **Макроархитектура** — видимая программисту модель: регистры, команды, прерывания (ISA — Instruction Set Architecture)
- **Микроархитектура** — внутренняя реализация: конвейер, кэш, предсказание ветвлений (скрыта от программиста)

## 5. Система команд и архитектуры процессоров

### Вопрос 14–18. Архитектуры CISC, RISC, MISC, VLIW

| Архитектура | Расшифровка | Характеристика | Примеры |
|-------------|-------------|----------------|---------|
| **CISC** | Complex Instruction Set | Много сложных команд, переменная длина, много адресаций | x86, 68k |
| **RISC** | Reduced Instruction Set | Мало простых команд фиксированной длины, load/store | ARM, MIPS, RISC-V, AVR |
| **MISC** | Minimal Instruction Set | Минимальный набор, часто 1 адрес, стековая модель | MU-1 (учебные), некоторые FPGA-soft |
| **VLIW** | Very Long Instruction Word | Несколько операций в одном длинном слове, компилятор планирует параллелизм | Itanium, TI C6000 DSP |

**CISC:** меньше команд в программе, сложное аппаратное декодирование.  
**RISC:** простое железо, быстрый конвейер, больше команд в программе.  
**VLIW:** перенос сложности в компилятор.

## 6. Представление чисел

### Вопрос 12. Знаковые и беззнаковые числа

**Беззнаковые (unsigned):** все биты — величина. Диапазон n бит: 0 … 2ⁿ−1.

**Знаковые (signed):** старший бит — знак (обычно дополнительный код).

**Коды:**
| Код | Описание | Пример 8 бит (−5) |
|-----|----------|-------------------|
| Прямой | Знак + модуль | 10000101 |
| Обратный | Инверсия модуля | 11111010 |
| Дополнительный | Обратный + 1 | 11111011 |

**Пример:** +42 = 00101010, −42 в доп. коде = 11010110.

BCD — двоично-десятичный код (8421): каждая декада отдельно.

## 7. Центральный процессор

### Вопрос 13. Центральный процессор (ЦПУ)

**ЦПУ** — центральное устройство ЭВМ, выполняющее команды программы.

**Структура:**
```
┌─────────────────────────────────────┐
│              ЦПУ                    │
│  ┌─────┐  ┌─────┐  ┌────────────┐  │
│  │ ОУ  │  │ АЛУ │  │ Регистры   │  │
│  └─────┘  └─────┘  └────────────┘  │
│  ┌─────────────────────────────┐   │
│  │ Устройство управления (УУ)  │   │
│  └─────────────────────────────┘  │
│  ┌──────────┐  ┌───────────────┐   │
│  │ Счётчик  │  │ Шинный        │   │
│  │ команд   │  │ интерфейс     │   │
│  └──────────┘  └───────────────┘   │
└─────────────────────────────────────┘
```

- **ОУ** — операндные устройства (регистры)
- **АЛУ** — арифметико-логические операции
- **УУ** — декодирование и управление
- **Счётчик команд (IP/PC)** — адрес следующей команды

## 8. Микропроцессорные системы

### Вопрос 19. Модульная организация

**Модульность** — система строится из стандартных функциональных блоков (МП, память, В/В, таймеры), соединённых магистралями. Модули можно заменять и расширять без переделки всей системы.

### Вопрос 20. Многофункциональность и специализация

Один физический модуль может выполнять разные функции (программно), но каждый модуль **специализирован** на своей задаче: МП — вычисления, контроллер памяти — память, UART — последовательный обмен.

### Вопрос 21. Магистральность

**Магистральность** — обмен информацией между модулями только через **общие шины** (магистрали), а не прямые связи «каждый с каждым».

### Вопрос 22. Магистраль

**Магистраль (шина)** — набор параллельных проводников для передачи данных, адресов и управляющих сигналов.

**Шины нижнего уровня:**
- **Шина данных** — данные и команды
- **Шина адреса** — адрес ячейки или устройства
- **Шина управления** — RD, WR, INT, CLK, сигналы готовности

### Вопрос 23. Микропрограммируемость

Сложные команды CISC реализуются как последовательность **микрокоманд** в управляющей памяти. Позволяет гибко менять поведение без перепроектирования железа.

### Вопрос 24. Регулярность структуры

Модули системы имеют **однотипную** структуру подключения к магистрали (одинаковый интерфейс, протокол, тайминги), что упрощает масштабирование и стандартизацию (ISA, PCI, USB).

## 9. Регистры (тема 3.6)

*Ниже — типичная модель на примере x86-подобных и общих принципов (на экзамене допустимо ссылаться на изучаемый МП).*

### Вопрос 25. Регистры общего назначения (РОН)

Хранят операнды и результаты. Примеры (x86): AX/EAX, BX, CX (счётчик), DX.  
16-битные МК: пары регистров. AVR: R0–R31. PIC: W-регистр + файловые регистры.

### Вопрос 26. Сегментные регистры

Указывают **базу сегмента** в памяти (модель сегментов x86): CS (код), DS (данные), SS (стек), ES, FS, GS. Смещение + база = линейный адрес.

### Вопрос 27. Регистр флагов (признаков)

Биты результата последней операции:

| Флаг | Значение |
|------|----------|
| ZF | Ноль |
| CF | Перенос |
| SF | Знак |
| OF | Переполнение |
| PF | Чётность |
| IF | Разрешение прерываний |
| DF | Направление строковых операций |

### Вопрос 28. Программная модель МП

Совокупность регистров и памяти, доступных программисту: РОН, специальные регистры, СК, указатель стека, флаги, иногда регистры управления периферией.

### Вопрос 29. Счётчик команд (IP/PC)

Хранит адрес **следующей** команды. Изменяется автоматически при выборке; при переходах — командами JMP, CALL, RET; при прерывании — сохраняется в стек.

## 10. Машинные команды и программы

### Вопрос 30. Машинная команда, операция, программа

- **Машинная операция** — элементарное действие (сложение, сдвиг)
- **Машинная команда** — закодированное указание МП выполнить операцию
- **Программа** — упорядоченная последовательность команд

**Классификация по операциям:** арифметические, логические, передача данных, переходы, ввод-вывод, управление системой.

### Вопрос 31. Машинный такт и машинный цикл

- **Машинный такт** — один импульс синхронизации
- **Машинный цикл** — время выполнения одной фазы команды (выборка, декодирование, исполнение)

**Структура простой команды:** [Код операции | операнд 1 | операнд 2 | результат]

### Вопрос 32. Структуры машинных команд

1. **Трёхадресная:** OP dest, src1, src2  
2. **Двухадресная:** OP dest, src  
3. **Одноадресная:** OP addr (аккумулятор)  
4. **Нульадресная:** стек (MISC)  
5. **Команды с непосредственным операндом**

### Вопрос 33. Структура программы на ассемблере

1. Заголовок / директивы (ORG, EQU, INCLUDE)  
2. Сегмент данных (переменные)  
3. Сегмент кода  
4. Точка входа (START)  
5. Подпрограммы  
6. Конец (END)

### Вопрос 34. Структура программы на языке C

1. Директивы препроцессора (`#include`)  
2. Объявления глобальных переменных  
3. Прототипы функций  
4. `main()` — точка входа  
5. Функции — модули кода  

**Функции** — инкапсуляция логики, повторное использование.  
**Комментарии** — пояснение для человека; компилятор их игнорирует.

## 11. Адресация (тема 4.3)

**Адресация** — способ указания местоположения операнда.

**Виды:** неявная, непосредственная, прямая (регистровая), косвенная, индексная, относительная.

### Вопрос 35. Неявная адресация

Операнд задаётся **неявно** правилами архитектуры (аккумулятор, стек). Пример: `INC A` — операнд всегда аккумулятор.

### Вопрос 36. Непосредственная

Операнд **в самой команде**. Пример: `MOV AX, 5` — число 5 в коде команды.

### Вопрос 37. Прямая (регистровая)

Адрес или операнд в **регистре** / по фиксированному адресу. Пример: `MOV AX, [1000h]` или `MOV R1, R2`.

### Вопрос 38. Косвенная

В команде указан регистр, содержащий **адрес** операнда. Пример: `MOV AX, [BX]` — BX указывает на память.

## 12. Память

### Вопрос 39. Ёмкость памяти

**Ёмкость** — объём хранимых данных.

**Единицы:** бит, байт (8 бит), Кбайт (1024 байт), Мбайт, Гбайт, Тбайт.

**Временные характеристики:**
- **Время доступа** — от запроса до появления данных
- **Время цикла** — минимальный интервал между обращениями
- **Пропускная способность** — байт/с

### Вопрос 40. Стековая память

**Стек** — область памяти с дисциплиной **LIFO** (Last In, First Out).  
Операции: PUSH (положить), POP (снять).  
**Указатель стека (SP)** — адрес вершины. Растёт вниз или вверх (зависит от архитектуры).  
Используется для подпрограмм, прерываний, локальных переменных.

## 13. Прерывания

### Вопрос 41. Прерывание

**Прерывание** — принудительная передача управления от текущей программы к **обработчику** при наступлении события.

**По источнику:**
- Аппаратные (внешние, внутренние)
- Программные (INT n, syscall)
- Исключения (деление на 0, page fault)

### Вопрос 42. Система прерываний

Совокупность аппаратуры и программ для обработки запросов.

**Характеристики:** число уровней, приоритеты, векторизация, маскирование, время отклика, вложенность.

```
Событие → Контроллер прерываний → МП сохраняет контекст →
→ переход по вектору → обработчик → восстановление → RETI
```

### Вопрос 43. Маскирование

**Маскирование** — запрет реакции на прерывание (бит IF, регистры маски).

- **Вектор прерывания** — адрес обработчика
- **Маска прерывания** — бит разрешения/запрета канала
- **Флаг прерывания** — признак ожидающего запроса

### Вопрос 44. Передача управления. Классификация внешних

При прерывании: сохраняются PC и флаги → вектор → обработчик → восстановление.

**Внешние прерывания:**
- **Маскируемые** — можно запретить (INTR в x86)
- **Немаскируемые (NMI)** — критические события, не блокируются обычной маской

---

## 14. Конвейер (тема 4.4)

### Вопрос 45. Конвейерная обработка

**Конвейеризация** — разбиение выполнения команды на **стадии** (выборка, декодирование, исполнение, запись), выполняемые параллельно для разных команд.

```
Такт:     1    2    3    4    5
Команда1: F | D | E | W |
Команда2:   | F | D | E | W |
Команда3:     | F | D | E | W |
```

Цель — CPI → 1, повышение тактовой эффективности.

### Вопрос 46. Конфликты конвейера

| Класс | Описание | Причина |
|-------|----------|---------|
| Структурные | Конкуренция за ресурс | Одна память для команд и данных |
| Данные | Зависимость по данным | Результат ещё не записан |
| Управления | Ветвление | Неизвестен адрес следующей команды |

Решения: bypassing, stall, предсказание ветвлений, раздельный кэш.

### Вопрос 64 (удалённый). Фиксированные и динамические приоритеты

**Фиксированные приоритеты:** каждому источнику прерывания назначен **постоянный** уровень. Старший всегда обслуживается первым. Простая реализация, предсказуемость.

**Динамические приоритеты:** приоритет **меняется** (например, по принципу «голодания» — кто дольше ждал, тот важнее). Гибче, но сложнее аппаратура.

## 15. Порты ввода-вывода и тактирование

### Вопрос 47. Порты ввода-вывода

**Порт В/В** — группа выводов МК для обмена с периферией.

**По типу сигнала:**
- Цифровые (0/1)
- Аналоговые (АЦП/ЦАП)

**По направлению:** вход, выход, двунаправленный.

### Вопрос 48. Выбор тактового генератора

- Требуемая частота и точность
- Диапазон температур
- Старт-время (кварц дольше RC)
- Потребление (низкочастотный кварц / внутренний RC)
- Наличие PLL
- Допуск и стабильность (UART, USB требуют точности)
- Стоимость и габариты

### Вопрос 49. Режимы тактирования PIC

- **LP** — низкочастотный кварц  
- **XT** — кварц/резонатор средней частоты  
- **HS** — высокочастотный кварц  
- **RC** — резистор-конденсатор (внутренний или внешний)  
- У современных PIC также INTOSC, PLL

### Вопрос 50. Подключение кварцевого резонатора

```
        ┌─────────┐
   ─────┤ OSC1     │
  │    │         │ МК
  │    │     OSC2├─────
  │    └─────────┘
  ╔═══╗
  ║ X ║  кварц / резонатор
  ╚═══╝
  │         │
 C1        C2   (нагрузочные 15–33 пФ, к GND)
```

Кварц между OSC1 и OSC2, конденсаторы к земле. Расстояние дорожек минимальное.

### Вопрос 51. Рекомендации по кварцу

- Ёмкость C1, C2 по datasheet (обычно 15–22 пФ для 8–20 МГц)
- Резонатор на номинальную частоту и тип (HS/XT)
- Не перегружать выводы (ESD, длина дорожек)
- Для ATmega: fuse CKSEL биты под источник тактирования

## 16. Микроконтроллеры производителей

### Вопрос 52. Atmel (Microchip) — AVR

**Семейства:** tinyAVR, megaAVR, XMEGA, AVR DA/DB.  
**Особенности:** RISC, Гарвард, Flash/EEPROM/SRAM, богатая периферия (UART, SPI, I2C, таймеры, АЦП).  
**Примеры:** ATmega328P (Arduino Uno), ATtiny85.  
**Разработка:** Atmel Studio / Microchip Studio, avr-gcc.

### Вопрос 53. Microchip — PIC

**Семейства:** PIC10, PIC12, PIC16, PIC18, PIC24, PIC32, dsPIC.  
**Особенности:** Гарвард, широкий диапазон, низкое потребление, MPLAB X.  
**Примеры:** PIC16F84A, PIC18F4550.

### Вопрос 54. Analog Devices

МК со встроенной **высокоточной аналоговой** периферией: 24-бит АЦП, DAC, измерительные усилители.  
Семейства: ADuC, MAX32 (после слияний). Для промышленных измерений.

### Вопрос 55. Intel

Исторически: 8051, 80186/80188 (встраиваемые).  
Современный фокус — процессоры Core/Xeon, а не массовые МК.  
8051-совместимые ядра до сих пор лицензируются во встраиваемых IP.

## 17. PIC16, AVR, JTAG, развитие МП

### Вопрос 56. PIC16

- 8-бит, Гарвард, 14/16-бит слово команды
- Flash программ, EEPROM данных, RAM
- Таймеры, порты, прерывания, Watchdog
- Простые и дешёвые, учебные проекты

### Вопрос 57. Память PIC16

- **Program memory (Flash)** — команды, отдельная шина
- **Data memory (RAM)** — регистры + ОЗУ; банки регистров
- **EEPROM** — энергонезависимые данные
- Адресное пространство данных и программ **разделено**

### Вопрос 58. Блоки AVR

| Блок | Функция |
|------|---------|
| Ядро CPU | Выполнение команд |
| Flash | Программа |
| SRAM | Данные, стек |
| EEPROM | Настройки |
| Порты GPIO | В/В |
| Таймеры/Счётчики | Задержки, ШИМ |
| UART/SPI/I2C | Связь |
| АЦП/Компаратор | Аналоговые сигналы |
| Watchdog | Сброс при зависании |

### Вопрос 59. JTAG

**JTAG** (IEEE 1149.1) — 4-проводной интерфейс: TDI, TDO, TCK, TMS (+ TRST).  
Используется для **внутрисхемной отладки** (останов, пошаг, просмотр памяти) и **ISP программирования** Flash. На AVR также debugWIRE (1 провод) на младших чипах.

### Вопрос 60. Направления развития МП

- Уменьшение техпроцесса (нм)
- Многоядерность и SIMD
- Снижение энергопотребления
- Интеграция периферии (SoC)
- ИИ-ускорители (NPU)
- RISC-V как открытая архитектура
- Безопасность (TrustZone, шифрование)

## 18. Схемотехника PIC и AVR

### Вопрос 61. PIC: кнопки, светодиоды, датчики

**Светодиод:** через резистор 220–470 Ом; катод на GND (активный HIGH) или анод на VCC (активный LOW).  
**Кнопка:** подтяжка (внутренняя weak pull-up или внешняя 10 кОм к VCC), антидребезг программный или RC (100 нФ).  
**Датчики:** цифровые — как кнопка; аналоговые — на вход АЦП (если есть) с согласованием уровня.  
Ток порта PIC ограничен (~20 мА на вывод) — не нагружать напрямую мощные устройства.

### Вопрос 62. AVR: кнопки, светодиоды, датчики

Аналогично PIC. ATmega328P:  
- `DDRx` — направление, `PORTx` — уровень / pull-up  
- Внутренняя подтяжка: `PORTx |= (1<<n)` при `DDR=0`  
- АЦП: опорное напряжение, делитель для датчиков  
- Для индикации — ток 5–15 мА через резистор
